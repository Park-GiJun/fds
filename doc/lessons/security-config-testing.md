# 보안 설정은 테스트가 곧 명세다

## 배운 점
SecurityConfig를 작성했지만 테스트가 없었다. 설정이 "동작한다"는 증거가 없는 것과 같다.
보안 설정 변경 시 회귀를 잡을 유일한 안전망이 테스트이므로,
보안 컴포넌트는 구현과 테스트를 동시에 완성해야 한다.

특히 Spring Security의 `authorizeHttpRequests`는 순서에 민감하여
미등록 경로의 기본 동작이 허용/차단 중 어느 쪽인지 테스트로 확인해야 한다.

## 적용 방법
SecurityConfig 작성 시 다음 테스트를 반드시 함께 작성:
- [ ] 인증 필요 엔드포인트에 미인증 접근 → 401 반환 확인
- [ ] 공개 엔드포인트 접근 → 200 반환 확인
- [ ] 미등록 경로 접근 → 403 또는 401 반환 확인
- [ ] 관리자 전용 엔드포인트에 일반 사용자 접근 → 403 확인
