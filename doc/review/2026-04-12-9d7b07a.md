# Code Review — Config Server fallback 제거 + Co-Authored-By 정리

- **커밋**: 9d7b07a, ea6e9c0
- **날짜**: 2026-04-12
- **기술 심각도**: MEDIUM
- **품질 심각도**: LOW

---

## 리뷰어별 요약

### 강현수 — Architect (LOW)

`optional:` 접두사 제거로 Config Server 의존성이 필수화되었다. 아키텍처 관점에서 MSA 인프라 설정이 단일 소스(fds-eureka-server의 classpath:/config)로 중앙화된 것은 긍정적이다. 단, fds-eureka-server가 Config Server 겸 Service Registry를 동시에 담당하는 구조이므로 이 단일 노드가 **SPOF(Single Point of Failure)**임은 명확히 인지해야 한다. 개발 단계에서는 수용 가능하나, 운영 전환 시 Config Server와 Eureka 분리가 필수다.

**발견 이슈**
- SPOF 구조 (Config Server + Eureka 동일 노드): 개발 수용, 운영 부채 등록 필요

---

### 박서진 — Security (LOW)

`optional:configserver:` 제거는 보안 측면에서도 개선이다. 이전에는 Config Server 미접속 시 로컬 fallback값(localhost 주소, 기본 포트)이 노출되어 공격 표면이 생길 수 있었다. 현재 application.yml에 남은 `${CONFIG_USERNAME:config}` / `${CONFIG_PASSWORD:config-secret}`의 기본값은 여전히 존재하지만 이는 이전 리뷰에서 이미 등록된 P1 부채다 (운영 프로파일 분리 필요).

**발견 이슈**
- 신규 없음. 기존 P1 부채 (`CONFIG_PASSWORD` 기본값 `config-secret`) 미해결 상태 유지.

---

### 이도윤 — Performance (INFO)

서비스 yml에서 삭제된 설정 중 `management.endpoints.web.exposure.include: health,info,prometheus`가 fds-eureka-server의 `classpath:/config/application.yml`에 전 서비스 공통으로 이미 존재함을 확인. 정상 이관이다. 성능 지표 노출에는 변화 없음.

**발견 이슈**
- 없음

---

### 정하은 — Code Quality (LOW)

5개 서비스 application.yml이 동일한 5줄 구조(`server.port`, `spring.application.name`, `config.import`, `cloud.config.username/password`)로 정규화되었다. DRY 원칙 달성, 가독성 최대화. ea6e9c0의 Co-Authored-By 제거도 커밋 메시지 일관성 측면에서 정당한 정리다.

**발견 이슈**
- `configserver:http://localhost:8761` 하드코딩이 5개 파일에 중복. 환경변수 `${CONFIG_SERVER_URL:http://localhost:8761}` 패턴으로 통일하면 docker/prod 환경에서 override 없이 기동 가능.

---

### 김태현 — Testing (LOW)

Config Server fallback 제거는 통합 테스트 전략에 영향을 준다. `@SpringBootTest` 기반 테스트가 Config Server 없이 기동을 시도하면 실패할 수 있다. 현재 테스트 코드(`fds-generator`, `fds-gateway`)가 Config Server를 mock하거나 `spring.config.import=optional:configserver:...`를 테스트 프로파일에서 override하고 있는지 확인이 필요하다.

**발견 이슈**
- P1: `@SpringBootTest` 슬라이스 테스트가 Config Server 의존으로 깨질 가능성 존재 — 테스트 프로파일에 `spring.config.import=optional:configserver:none` 또는 `spring.cloud.config.enabled=false` 설정 필요.

---

### 윤지아 — Domain Expert (INFO)

generator.target-url fallback 제거는 GeneratorService의 Ktor Client 대상 URL이 Config Server에서 주입됨을 의미한다. FDS 비즈니스 로직 관점에서 트랜잭션 생성 대상이 코드에서 설정으로 이동한 것은 적절하다. 도메인 로직 변경 없음.

**발견 이슈**
- 없음

## Tech Lead 판정 — 최민준

**기술 심각도: MEDIUM**

이번 변경의 핵심은 의도적인 Fail-Fast 전략으로, 아키텍처 결정으로서 타당하다. 다만 두 가지 구조적 위험이 병존한다.

**1. SPOF 인지 등록 (Critical 부채 후보)**
- fds-eureka-server(8761)가 Config Server + Service Registry를 겸하는 구조.
- 전 서비스가 기동 시 이 단일 포트에 의존하므로, 해당 프로세스 장애 시 전 서비스 기동 불가.
- 개발 환경에서는 수용 가능하나 운영 전환 전 분리 필수. Tech Debt에 등록.

**2. 테스트 기동 안정성 (P1)**
- `spring.config.import: configserver:...` (optional 제거)로 인해 CI/테스트 환경에서 Config Server 없이 `@SpringBootTest`가 실행되면 즉시 실패.
- 테스트 리소스에 `spring.config.import=optional:configserver:none` override 또는 `spring.cloud.config.enabled=false` 설정이 없으면 테스트 파이프라인 붕괴 가능.
- **Action Item P1으로 등록.**

**반복 실수 추적**
- `CONFIG_PASSWORD` 기본값(`config-secret`) 운영 프로파일 미제거 — 2회 이상 지적. 다음 커밋 전 해결 권고.
