### 2. 박서진 — Security
**잘한 점**
- 평문 저장 경로 차단 (#86 merged) ✅
- Logback %mask 컨버터 추가 — 로그 경로 1차 방어
- lat/lon 범위 검증 — 지오펜스 우회 차단

**취약점**
- `logback-spring.xml` 교체 시 기존 설정 손실 의심 (FILE appender, framework logger levels, springProfile blocks — 에이전트 보고서에서 "overwritten" 명시). **운영 로깅 구성 regression 가능성**. **[HIGH — regression 우선 확인]**
- `CardNumberMaskingConverter`는 `formattedMessage`만 마스킹. **예외 스택 트레이스, MDC, structured args는 미커버**. 예외 메시지에 PAN이 포함되면 그대로 노출. **[MEDIUM]**
- `PassthroughCardEncryptor`가 master에 존재 → 개발/스테이징 DB에 실제 PAN 유입 시 PCI-DSS 위반 그대로. 운영 정책(synthetic PAN only) 명시 필요. **[HIGH — 운영]**
- `CardEncryptor.decrypt` 현재 무호출. 후속 개발자가 편의 조회 경로를 추가하면 평문 복호화 경로 부활. YAGNI 원칙으로 제거 또는 접근 제어. **[LOW]**
- GlobalExceptionHandler가 예외 `message`를 응답 본문에 담음. `DomainValidationException`의 메시지에 민감 정보가 들어갈 수 있음. Bean Validation은 필드 에러가 직렬화될 수 있어 **입력값이 응답 메시지로 echo될 위험**. **[MEDIUM]**

**심각도**: HIGH / **이슈**: 5건

### 3. 이도윤 — Performance
**잘한 점**
- exists+save 2단계 제거 → unique constraint catch (race 해소)
- Clock 주입 → 테스트 격리

**이슈**
- `CardEncryptor.encrypt`는 save마다 호출. Passthrough는 비용 0이나 KMS 전환 시 ~10-20ms/call. 10K TPS 목표 고려 시 **key caching + 비동기 배치** 설계 필요. **[MEDIUM — 후속]**
- `CardNumberMaskingConverter` 정규식이 모든 로그 라인에서 실행됨. 고로그 서비스에서 CPU 오버헤드 발생 가능 — 구조화 로깅(JSON) 또는 MessagePatternFilter로 제한 권장. **[LOW]**

**심각도**: MEDIUM / **이슈**: 2건
